# SoccerStat

Тестовое приложение для просмотра футбольных лиг, команд и календарей матчей.

Стек:
- React 19
- TypeScript
- Vite
- MUI
- TanStack Query
- React Router

Реализовано:
- страница лиг с поиском и клиентской пагинацией;
- страница команд с поиском и клиентской пагинацией;
- календарь лиги с хлебными крошками, фильтром по диапазону дат и пагинацией;
- календарь команды с хлебными крошками, фильтром по диапазону дат и пагинацией;
- локализация интерфейса на русский;
- перевод времени матчей из UTC в локальное время пользователя;
- обработка ошибок API и превышения лимита запросов.

## Запуск

1. Установить зависимости:

```bash
npm install
```

2. Создать `.env.local` по образцу `.env.example`.

3. Запустить приложение:

```bash
npm run dev
```

4. Собрать production-версию:

```bash
npm run build
```

## Переменные окружения

Для локальной разработки рекомендуется использовать прокси Vite, тогда токен не попадёт в клиентский бандл.

Основной сценарий:
- `FOOTBALL_DATA_TOKEN` — токен `football-data.org`, который Vite-прокси подставляет на серверной стороне.
- `FOOTBALL_DATA_API_URL` — базовый адрес API, по умолчанию `https://api.football-data.org`.
- `VITE_API_BASE_URL` — адрес, по которому клиент обращается к API. По умолчанию `/api/football-data`.

Опционально:
- `VITE_FOOTBALL_DATA_TOKEN` — клиентский токен для прямых запросов без прокси. Использовать только если вы осознанно принимаете, что токен будет виден в клиенте.

## Замечания

- `football-data.org` имеет ограничения бесплатного тарифа. При исчерпании лимита приложение показывает понятное сообщение пользователю.
- Для статического деплоя нужен прокси или serverless-обёртка перед `football-data.org`, если вы не хотите публиковать токен в клиенте.
